# ModelForge Template References

These reference files describe the data contracts available to future generated templates.

Future templates should reuse these references in comments, inline guidance, and JSDoc blocks so developers do not need to rely on `console.log` just to discover what a context object contains.

These references use static examples first. Real runtime snapshots may be added later when ModelForge starts generating concrete files from captured project state.

The examples in this folder are representative and may vary by model, issue, or evaluation structure. They are intended to be practical, navigable examples, not final generated code.

These files do not generate anything by themselves. They are source references for future template generation and template specs.
